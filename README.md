# Packet Forgery and Intrusion detection system
This project presents the design and implementation of an integrated, high-performance network security solution combining low-level packet manipulation with real-time threat intelligence gathering. Traditional network defenses rely on passive packet-dropping mechanics, which often fail to actively disrupt automated adversarial recon tools. This system addresses that limitation by functioning as an active network "Tarpit" and intrusion detection engine.  Using a decoupled, multi-threaded Python architecture, the backend application utilizes the scapy library to monitor network interface traffic via raw sockets. A heuristic detection engine continuously parses Layer 3 and Layer 4 headers to isolate malicious traffic anomalies, specifically identifying multi-port scanning behavior (e.g., Nmap reconnaissance). Upon detection, the packet forgery engine dynamically generates and injects spoofed TCP SYN-ACK responses back onto the network wire. This defensive deception tricks scanning mechanisms into identifying closed ports as open, effectively exhausting the attacker’s computational resources and disrupting their scanning sequence.  To ensure strict data persistence, the system normalizes and logs all attack telemetry—including source IP tracking, geographic origin data, and targeted port metrics—into a relational database using parameterized SQL queries. The front-end consists of a native desktop application built via the tkinter framework. By employing asynchronous database polling mechanics (utilizing the event-driven .after() method), the GUI dynamically refreshes real-time attack logs and counters without blocking or degrading the execution speed of the core packet-sniffing backend.  Core Technical Pillars & Deliverables Low-Level Network Engineering: Implementation of Deep Packet Inspection (DPI), Berkeley Packet Filtering (BPF), and custom TCP/IP handshake manipulation utilizing scapy.  Database Architecture: Implementation of a relational schema designed for high-volume data logging, secured against SQL injection vulnerabilities.  Concurrent Software Design: Separation of the resource-heavy network sniffer from the user interface using a decoupled architecture to prevent application threading deadlocks.  Graphical User Interface (GUI): A native desktop application built with tkinter to provide system administrators with live, event-driven threat telemetry and monitoring controls.
